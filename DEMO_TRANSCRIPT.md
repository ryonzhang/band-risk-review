# Live Band demo — verified run

Captured from a real Band room (`app.band.ai`) with the four agents running live
(`python band_agents.py data|quant|calibration|reviewer`) and powered by Gemini.
Each question is posted to the room mentioning `@Risk-Data`; the agents hand off
through the room: **Data → Quant → Calibration → Reviewer → user**. The Reviewer
narration is generated by Gemini; every number comes from `risk_tools.py`.

The four scenarios reproduce the four honest outcomes:

| Ask | Verdict | Why (from the Reviewer's message in the room) |
|---|---|---|
| Should I add to my BTC position? | **APPROVED** (size 0.1141) | No compliance breaches; supported by a 0.7283 calibrated probability. |
| ETH looks hot on socials — should I add? | **ABSTAIN_UPHELD** (0.0) | Confidence 0.1529 below the 0.3 floor — conflicting signals. |
| Thinking about a DOGE position? | **ABSTAIN_UPHELD** (0.0) | No data available for the subject asset. |
| Add more SOL here? | **ESCALATE** | Drawdown contribution of 9% of NAV breaches the 6% gate → human risk officer. |

## Reproduce it

```bash
# 1. agents (4 processes) — see SETUP_BAND.md for creds in agent_config.yaml + .env
python band_agents.py data &
python band_agents.py quant &
python band_agents.py calibration &
python band_agents.py reviewer &

# 2. in a Band room with all four agents added, post:  @Risk-Data Should I add to my BTC position?
#    (or drive it headlessly with the helper below, BAND_ROOM_ID in .env)
python post_question.py "Should I add to my BTC position?"
python read_transcript.py reviewer        # print the room's audit trail
```

The offline scaffold (`run_demo.py`, `test_workflow.py`, 16/16) reproduces the same
four verdicts with no network/credentials.
